Om arbetet

Den här uppgiften gick ut på att bygga en fristående app som hämtar data från ett API.

Appens funktioner består av att visa, radera och uppdatera data om arbetserfarenheter, vilket skulle kunna vara bra för ett framtida CV.

Javascripts fetch används för att hämta data från ett REST-API på en annan domän. API-appen har en SQLite-databas för lagring av poster och kan hantera CRUD-operationer på en datatyp, arbetserfarenheter.

Front-end och API har publicerats på olika servrar, via Render och Netlify, och CORS används för säker kommunikation mellan dem.

För applikationer där data används på flera ställen kan ett API vara en bra lösning, och den här uppgiften gav en liten smak av hur det skulle kunna vara att skapa en tjänst som används av flera appar.

Arbetet med fetch och API-routes var inte särskilt komplicerat, så längre jag inte skrev fel på adresserna.
